# Skill 作成ガイド

## 概要

Skill は「タスク特化型の Claude 専用マニュアル」。CLAUDE.md との違いは読み込みタイミング。

| | 読み込み | 用途 |
|---|---|---|
| CLAUDE.md | セッション開始時に全文 | 常時参照の全体ルール |
| Skill | 起動時は `name` と `description` のみ。選択時に本文を読み込む | 必要時だけの専門知識 |

## description の書き方（最重要）

Skill の効果は **description で9割決まる**。Claude の判断材料は description のみで、本文はトリガーに影響しない。

### 原則

- **日本語で書く**
- 「何をするか」+「いつ使うか」の2要素を必ず含める
- トリガーキーワードを網羅する
- 三人称で書く（システムプロンプトに注入されるため）
- 本文ではなく description にトリガー情報を集約する

### テンプレート

```yaml
description: |
  [動詞]+[対象]+[基準]。[具体処理2-3文]。
  [条件1]・[条件2]・[条件3]のときに使用する。
```

### 良い例・悪い例

| 悪い例 | 良い例 |
|--------|--------|
| `Worktree を作成する` | `Worktree を作成し指定ブランチを checkout する。ブランチ名を引数として受け取る。worktree の作成・ブランチの隔離作業・新しい feature ブランチでの作業開始時に使用する。` |
| `コードレビューする` | `チーム規約に基づくコードレビューを実施する。命名・エラー処理・テストカバレッジ・アーキテクチャ準拠をチェックする。コードレビュー・PR レビュー・コード品質の評価時に使用する。` |

## 本文（SKILL.md）の書き方

- 500行以下に収める
- 詳細は `references/` に分離し、本文は「目次 + コア手順」に絞る
- 1つの Skill は単一責務にする（万能 Skill は description が曖昧になり選択されない）

## ファイル構成

```
.claude/skills/
└── <skill-name>/
    ├── skill.md          # フロントマター + 本文
    └── references/       # 必要に応じて補足資料
```

## フロントマター

```yaml
---
name: <skill-name>
description: |
  <description>
---
```

その他のフロントマターオプション:

- `context: fork` — Skill を別コンテキストで実行する
- `agent: <agent-name>` — 特定のエージェントに委譲する
- `disable-model-invocation: true` — 明示的呼び出し専用にする（デプロイ等）

## 作成フロー

1. まず手動で動かして理想のワークフローを確立する
2. 成功したら Skill 化する（「動くものを先に作り、後から Skill 化する」が最短ルート）
3. description を設計パターンに沿って書く
4. テストして Claude が自動で呼び出すか確認する
